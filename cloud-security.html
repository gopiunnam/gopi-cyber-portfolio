Overview

I developed practical best-practice guidelines for securing AWS and Azure environments, focused on identity security, logging, monitoring, and risk reduction. This project summarizes a repeatable approach for improving cloud posture using security controls aligned with common frameworks.

Key Focus Areas

  • Identity & Access: IAM least privilege, MFA, conditional access, role-based access control
  • Logging & Monitoring: AWS CloudTrail, Azure logging, centralized alerting and detection
  • Secure Configuration: CIS benchmarks, hardened baselines, secure defaults
  • Threat Detection: Security Hub / Azure Security Center alerts and investigation workflow
  • Vulnerability & Patch: prioritization strategy and remediation tracking

Tools Mentioned

  • AWS Security Hub, CloudTrail
  • Azure Security Center (Defender), Sentinel (where applicable)
  • Entra ID (Azure AD), IAM

Outcome

  • Created a structured checklist for assessing and improving cloud security posture.
  • Enabled repeatable security reviews for identity, logging, and monitoring gaps.
  • Produced documentation that can support audit readiness and continuous improvement.
